# DAY-11 - Intermediate-commands (فارسی)

### 🧠 Mechanism
- **پردازش خط لوله‌ای (Pipeline Filtering):** دستورات میانی لینوکس مانند یک جعبه ابزار ماژولار طراحی شده‌اند. هدف اصلی در لایهٔ مکانیزم این است که ورودی/خروجی استریم‌ها (`stdin`/`stdout`) به کمک کاراکتر پایپ (`|`) بین این برنامه‌ها دست به دست شود تا یک دیتای خام به یک خروجی فیلترشده و منظم تبدیل گردد.

### 🔍 OS Hook
- **موتورهای متادیتا و ایندکس سیستم:** دستوراتی مثل `file` با خواندن کدهای جادویی (Magic Bytes) در هدر فایل با لایه فایل‌سیستم تعامل دارند، و ابزاری مثل `locate` به جای اسکن زندهٔ هارد، از یک دیتابیس ایندکس‌شدهٔ محلی استفاده می‌کند.

### 💻 Commands Reference

| دستور (Command) | سوییچ‌ها (Options) | توضیح خلاصه کاربرد (Summary) |
| :--- | :--- | :--- |
| **`grep`** | `-i` (حساس نبودن به حروف بزرگ و کوچک) <br> `-v` (معکوس کردن فیلتر) <br> `-E` یا `egrep` (استفاده از رگرس پیشرفته) | جستجو و استخراج خطوط حاوی یک عبارت خاص از فایل یا استریم ورودی. |
| **`less`** | `/word` (جستجو به جلو) <br> `?word` (جستجو به عقب) <br> `space` (صفحه بعد) / `b` (صفحه قبل) / `q` (خروج) | صفحه‌بندی و پیمایش آسان در فایل‌ها یا خروجی‌های بسیار بزرگ سیستمی. |
| **`pwd`** | `-P` (نمایش مسیر واقعی فیزیکی بدون در نظر گرفتن سیم‌لینک‌ها) | چاپ دایرکتوری کاری فعلی پروسه. |
| **`diff`** | `-u` (فرمت یکپارچه و استاندارد برای ابزارهای خودکار) | مقایسه خط به خط دو فایل متنی و استخراج تفاوت‌های آن‌ها. |
| **`file`** | بدون سوییچ خاص | بررسی هدر فایل برای تشخیص نوع و فرمت واقعی فایل (بدون وابستگی به پسوند). |
| **`find`** | `dir -name 'pattern' -print` | جستجوی زنده و واقعی در درخت دایرکتوری‌ها بر اساس نام (نیاز به کوتیشن برای ستاره). |
| **`locate`** | بدون سوییچ خاص | جستجوی آنی و فوق‌العاده سریع فایل‌ها از طریق دیتابیس ایندکس سیستم (mlocate). |
| **`head`** | `-n` (مشخص کردن تعداد خطوط از ابتدا) | نمایش خطوط ابتدایی یک فایل یا استریم دیتای خام (پیش‌فرض ۱۰ خط). |
| **`tail`** | `-n` (مشخص کردن تعداد خطوط از انتها) <br> `+n` (شروع نمایش از خط nام تا انتهای فایل) | نمایش خطوط انتهایی یک فایل یا استریم دیتای خام (پیش‌فرض ۱۰ خط). |
| **`sort`** | `-n` (مرتب‌سازی بر اساس ارزش عددی) <br> `-r` (معکوس کردن ترتیب مرتب‌سازی) | مرتب‌سازی متنی یا عددی خطوط موجود در یک فایل یا استریم. |

### 🛠️ Lab & Tools
**تست عملی: فیلترینگ و استخراج اطلاعات متنی با پایپ لاین**

در این آزمایش، سناریوی ترکیبی زیر را در اسکریپت `demo.sh` پیاده‌سازی می‌کنیم تا هماهنگی این دستورات را با هم ببینیم:

1. **تست فیلترینگ ترکیبی:** خروجی فایل سیستم را با `grep` فیلتر کرده و اولین و آخرین خطوط آن را با `head` و `tail` استخراج می‌کنیم.
2. **تشخیص هویت فایل:** از دستور `file` برای مچ‌گیری از فرمت یک فایل بدون پسوند استفاده می‌کنیم.
3. **تست تفاوت فایل‌ها:** دو فایل متنی کوچک با تفاوت جزیی ساخته و خروجی `diff -u` را بررسی می‌کنیم.


**کارگاه بررسی تفاوت دستور locate و find :**

1. یکی از فایل های قدیمی تر سیستم را انتخاب کنید
2. از دستور locate برای پیدا کردن آن استفاده کنید

```bash
locate FILE
```
3. حالا از دستور find استفاده کنید
```bash
find DIR -name file -print
```
4. در این مرحله یک فایل جدید ایجاد کنید
```bash
touch new_file
```

5. مجدد از دو دستور locate و  find  استفاده کنید.

خواهید دید که اینبار دستور locate کاربردی ندارد . زیرا فایل جدید شما برای ایندکس زدن زیادی جدید است.


### ⚠️ Pitfalls
- **تلهٔ داده‌های قدیمی در `locate`:** اگر فایلی را همین چند ثانیه پیش ساخته باشید، دستور `locate` آن را پیدا نمی‌کند؛ چون دیتابیس سیستم هنوز به‌روز (Update) نشده است. در این حالت باید از `find` استفاده کرد.
- **فراموشی کوتیشن در `find`:** اگر در دستور `find` برای الگوهایی مثل ستاره از تک‌کوتیشن استفاده نکنید (مثلاً بنویسید `-name *.txt`)، شل قبل از اجرای دستور آن را بسط داده و باعث بروز خطای سینتکس در ابزار `find` می‌شود.
