# DAY-14 - IO-redirection-and-error-anatomy (English)

### 🧠 Mechanism
- **Standard Streams Abstraction:** Every Linux process initializes with three canonical file descriptors: `0` for Standard Input (`stdin`), `1` for Standard Output (`stdout`), and `2` for Standard Error (`stderr`).
- **The stdout vs stderr Separation:** Utilities route successful operational logs to Channel 1, while tracking faults and routing diagnostics directly to Channel 2. Consequently, a bare `>` redirection operator intercepts only Channel 1, letting faults bypass the filter and spill onto the terminal screen.
- **Stream Merging Mechanics (`2>&1`):** This explicit syntax instructs the shell environment to bind the output configuration of Channel 2 (`stderr`) directly onto the existing redirection path of Channel 1 (`stdout`).



### 🔍 OS Hook
- **Kernel Interception of Memory Violations:** Memory failures like a `Segmentation fault` are not generated by the user-space binary text logs. When a process attempts to read or write to an unmapped or unallocated RAM physical address, the hardware Memory Management Unit (`MMU`) raises a fault trap, forcing the Linux kernel to abruptly terminate the culprit process via a `SIGSEGV` signal.

### 💻 Commands Reference

#### I/O Redirection Operators & Control Flags
| Operator / Command | Operational Summary |
| :--- | :--- |
| **`>`** | Redirects `stdout` (FD 1) to a target destination file, clobbering pre-existing data. |
| **`>>`** | Appends `stdout` streams onto the trailing end of a file without destroying existing bytes. |
| **`2>`** | Isolates and redirects the standalone error stream (`stderr` / FD 2) into a dedicated file. |
| **`> file 2>&1`** | Merges both `stdout` and `stderr` streams, channeling them bound together into one file. |
| **`<`** | Channels the raw data of a target file directly into a program's standard input (`stdin` / FD 0). |
| **`set -C`** | Enforces the noclobber parameter in Bash to block accidental file overwrites via `>`. |

#### Anatomy of System Hit Parade Errors
| Error String | Core Underlying Structural Cause |
| :--- | :--- |
| **`No such file or directory`** | The requested file descriptor path cannot be resolved anywhere across the active VFS layer. |
| **`Not a directory`** | A physical file node was erroneously processed as a directory boundary container along a path string. |
| **`Permission denied`** | The process lacks appropriate privilege bits (r/w/x) or clear security context mappings for the target. |
| **`Segmentation fault`** | The active process attempted an illegal physical memory segment violation, triggering a kernel shutdown. |

### ⚠️ Pitfalls
- **Accidental Clobbering via `>`:** Blindly using the `>` symbol instead of the `>>` append syntax completely wipes out the target file's historical data without warning blocks.
- **Chasing Secondary Cascading Faults:** When analyzing script runtime crashes, subsequent errors are typically side effects of the primary root issue (e.g., a missing primary configuration file spawning hundreds of allocation warnings). Always isolate and fix the earliest reported error first.
